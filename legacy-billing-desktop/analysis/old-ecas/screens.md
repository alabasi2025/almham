# تحليل شاشات تطبيق ECAS القديم

## المصدر

- ملف التطبيق: `D:\almham\imports\ECAS-clean-install\Electricity Customers Accounts System.exe`
- قاعدة التحليل: SQL Server `Ecas2673`
- الجداول المستخدمة: `FormData`, `FormEvent`, `Event`, `UserData`, `RankUser`
- ملف النصوص المستخرجة سابقاً من EXE: `D:\almham\imports\arabic-strings.txt`

## الخلاصة

- إجمالي السجلات في FormData: 171
- الشاشات الفعالة ذات Frm_ID > 0: 169
- الشاشات المعطلة/المخفية ذات Frm_ID < 0: 2
- الشاشات المباشرة بدون قائمة رئيسية (#): 12
- أحداث الصلاحيات المستخرجة من Event: موافق، إضافة، تعديل، حذف، بحث، طباعة، تفعيل، توقيف، ترحيل، فك الترحيل، إقفال، إحتساب، إلغاء الإحتساب، إستيراد، تصدير، إتصال / إرتباط، إرسال، إخلاء، إحلال، مزامنة، إعتماد

## ملاحظات عن شكل التطبيق القديم

- التطبيق قديم من عائلة VB6/COM ويعتمد على OCX مثل `MSCOMCTL.OCX`, `MSDATGRD.OCX`, `MSFLXGRD.OCX`, `TABCTL32.OCX`.
- بنية الواجهة قائمة رئيسية كبيرة + شاشات MDI/نماذج بيانات + صلاحيات لكل شاشة ولكل حدث.
- شاشة الدخول موجودة كعنصر رقم `9901`، وتستخدم `UserData.Us_Name` و`UserData.Us_PassWord`.
- التطبيق يربط الصلاحيات عبر `UserPrivileg` ويفتح الشاشات حسب `FormData` و `FormEvent`.
