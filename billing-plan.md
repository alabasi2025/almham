# 🧾 خطة نظام الفوترة Web المستقل (ECAS → Almham Billing)

> **الهدف**: استنساخ وظيفي كامل (174 شاشة) لنظام ECAS، بنفس المصطلحات والـ Workflow،
> مع واجهة عصرية (Angular 21 + PostgreSQL + Hono).

**مصدر البيانات**: `SQL Server (ECASDEV)` — قواعد Ecas2673 (الدهمية) + Ecas2668 (الصبالية/جمال/غليل)
**الوجهة**: `PostgreSQL (almham_billing_db)` — يُعاد الاستيراد دورياً

**حد المشروع**: مشروع مستقل داخل `billing-web/`، وليس جزءاً من شاشة دخول نظام العباسي الرئيسي.

## 🛠️ التقنية (ثابتة)

| الطبقة | التقنية |
|--------|---------|
| Frontend | Angular 21 Standalone + Signals + RTL |
| UI | Angular Material + Chart.js + Leaflet |
| Backend | Hono 4 + TypeScript 5 |
| ORM | Drizzle 0.38 |
| DB (نظامنا) | PostgreSQL 15+ |
| DB (مصدر ECAS) | SQL Server (ECASDEV) — read-only |
| Auth | JWT + HttpOnly cookies + bcrypt |

**قاعدة فصل:** لا يعتمد هذا المشروع على Angular/Hono الخاص بنظام العباسي الرئيسي، ولا يُدفن داخل `legacy-billing-desktop/`.

## 📋 الكود المرجعي للشاشات

نستخدم **أرقام ECAS الأصلية** (Frm_ID) في:
- مسارات URL: `/billing/f/9010` ← شاشة 9010
- أسماء المكوّنات: `Form9010MeansComponent`
- جدول `permissions`: `screen_code = 9010`

هذا يسهّل الربط مع وثائق ECAS والتدريب.
